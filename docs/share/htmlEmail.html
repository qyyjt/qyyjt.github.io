# html邮件编写

# 1. Doctype

目前，兼容性最好的Doctype是XHTML 1.0 Strict，事实上Gmail和Hotmail会删掉你的Doctype，换上这个Doctype。

<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">
<html xmlns="http://www.w3.org/1999/xhtml">
</html>

# 2. 布局

使用table布局，最好使用img,a ,span标签，通过table嵌套的方式，实现网页的布局。


<table border="0" cellpadding="0" cellspacing="0" width="1016" align="center" style="background-color: #fff;">
   <tbody>
       <tr>
           <td width='100%'>
              <table border="0" cellpadding="0" cellspacing="0" align="center"
              width='1016' style="background-color:#fff;table-layout: fixed; ">
                   <tbody>
                       <tr height='24'>
                            </tr>
                   </tbody>
              </table>
            </td>
        </tr>
    </tbody>
</table>

# 3. 图片

  • 在某些邮箱中图片不是默认加载的，往往加载前需要用户的许可 。

  • 图片不要使用base64图片，在outlook将会不展示。

 background-image: url(data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAAAaQAAAJECAYAAABZ4FAqAAAABGdBTUEAALGPC/
  • 引用图片使用img标签，style 内容里面 background 可以设置 color，但是 image 会被过滤。
 使用img标签
 <img border="0" style="display:block;outline:none;text-decoration:none,-ms-interpolation-mode:bicubic;border:none;"> 

背景色与背景图同时设置，保证图片不生效的情况下，背景图能展示
<table align="center" border="0" cellpadding="0" cellspacing="0" width="1120" height="185" style="background-color: #2F8DFD;background-image: url('https://cdn.finchina.com/app/sendMail/background@2x.jpg')">

# 4 .行内样式

  • css 只有内联样式才能兼容客户端与浏览器邮箱，而外部的style及head标签中的style有可能会客户端被删除。
<span style=" font-size: 24px;font-family: Arial, sans-serif;color: #fff; margin-left: 12px;">企业预警通</span>
  • css 不支持简写。
　想要表达
　 <p style="margin: 1em 0;">
　要写成下面这样
　<p style="margin-top: 1em; margin-bottom: 1em; margin-left: 0; margin-right: 0;">
　

# 5. js

html 邮箱不能引入js,是为了防止某些攻击。

# 6. 字体

在 HTML 邮件中，font-family 只支持系统字体(font-family: Arial, sans-serif; )，不支持自定义字体。

 <span style=" font-family: PingFangSC, PingFangSC-Light;color: #f7fbff;font-size: 13px;line-height: 18px;display: inline-block;">本报告产生于2021-05-01 08:52:30</span>

# 7. 行高

在 OutLook 中会有个默认的行高最小值，特别是当设置 font-family 为微软雅黑时，默认的行高差不多为 Word 中的两倍行距，如果 line-height 设置的值小于默认的行高，无论你设置的是多少，则始终使用默认值，在很多情况下这是不能忍的，好在有个神奇的 mso-line-height-rule，使用行高时添加 mso-line-height-rule:exactly; 就能使行高始终等于我们所设置的值。

<td style="mso-line-height-rule: exactly; line-height: 36px;">
    <!-- ... -->
</td>

# 8. 全局规则

  • 如果需要邮件居中显示，请在table里设定align="center"。
 <table align="center" border="0" cellpadding="0" cellspacing="0" width="1120" height='185' </table>

  • 若邮件模板内侧边或者上下有空白间距，不要用 padding，必须得用标准的 td 来设定空白间距，否则会导致各个邮箱解析不同。

      <tr align="center" height='15'>
    <td></td>
   </tr>
  <tr align="center" height='42'>
    <td> </td>
   </tr>

  • 不要写style标签、不要写class，所有CSS都用style属性，什么元素需要什么样式就用style写内联的CSS。

  • 大多数css3新技术不能使用，如 阴影，渐变色，hover事件，定位，只能使用常用的属性width,height,line-height,color。

  • 在td 里设置 margin 是无效的，不论是 margin-left、margin-right、margin-top 或者 margin-boottom 都没有效果。

  • outlook对背景图片不识别，只能只用img。

  • 少用float, margin,padding. 绝对定位不能用，清除浮动用<table style="clear:both"></table>。

  • 少用图片，邮箱不会过滤你的img标签，但是系统往往会默认不载入陌生来信的图片。
