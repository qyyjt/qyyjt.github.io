# 开发环境说明

项目在开发时会对应多个开发环境，不同的环境会映射不同分支的代码

环境 域名 分支 说明
dev 开发环境 appdev.finchina.com dev 开发环境
test 测试环境 apptest.finchina.com master 当在 test 分支发生过代码变动时，内网编译器会自动将 test 分支的代码编译并发布到 test 域名
uat 环境 appuat.finchina.com - 预发布环境
华为云 环境 apphwy.finchina.com - 外网发布双机房域名
外高桥 环境 appsh.finchina.com - 外网发布双机房域名
外网 环境 app.finchina.com - 发布完毕，外网域名

# dev 环境

我们在内网提交 dev 环境是，需要手动打包代码，然后把打包后的代码提交到 dev 环境对应的 svn 地址 (opens new window) ，内网编译器会自动将 svn 上的打包后的代码，推送到 dev 环境，一般在提交到 svn 10 分钟左右可以看到效果。

大部分项目都可以使用 gulp 自动构建部署，你只需要修改 gulpfile.js 这个配置文件中的内容，然后运行 gulp 命令，就可以打包、将打包后的代码更新到 svn，并且更新到 dev 环境的服务器，这样就可以立即在浏览器看到效果；若个别无法使用 gulp，可以使用用手动提交的方法。

TIP

若 gulp 之后发现源码文件被删除了，可能是 TortoiseSVN 安装的时候没有全部勾选，需重装 TortoiseSVN

通常情况，dev 环境的东西是比 test 环境和其它环境的功能多的，因为里面包括很多还没有发布出去的需求。

运行 gulp 命令之前需要修改下 gulpfile.js 中的提交日志，并确保各个目录正确

如果 gulp 后在 appdev 域名下久久看不到代码生效， 可能是以下原因之一造成的：

  1. gulp 时候报错，可查看 gulp 日志
  2. 输入代码的 dist 目录和配置文件中的名称不一致

或者可以直接登陆 dev 环境的服务器，进入代码对应的目录，检查服务器上的文件是否被更新了

# test 环境

相较于 dev 环境，test 环境是一个比较稳定的环境，因为它只有在项目升级时，才会提交代码。你可以把 test 环境看做是一个早期的预发布环境。

在项目需要发布新版本时，可在禅道看板 (opens new window)中查看已经定版的需求，

若我们的源码在git上，会将定版的需求集中提交到 master 分支。推送的方式也异于 dev 的 push 方式，你须要使用 merge request (opens new window) 的方式请求合并，在请求合并之后需要联系升级相关负责人完成提交合并。

若我们的源码在svn上，提交的方式和 dev 环境开发时的手动提交方式相同，就是把打包后的代码，提交到 test 环境对应的 svn 地址 (opens new window)；内网编译器会自动将 svn 上的打包后的代码，推送到 test 环境，一般在提交到 svn 10 分钟左右可以看到效果；提交之后记得在企业预警通APP中切换到 test 环境自测

# uat 环境

uat 环境是一个预发布环境。

一般进入这个发布环境之前，都会有升级单走升级流程。之后，将 test 环境的编译包升级到该环境。

进入到这个环境就意味着版本趋于稳定了，不会存在较大的改动，可能会有一些小的 bug 和样式修复；若又需要修复的问题，修复之后，需要联系项目经理或负责人重发 uat

# 华为云和外高桥

预警通线上有两个机房互相切换，即“双活”，这里的两个机房分别是华为云和外高桥。

当升级外网时，为了不影响外网用户的使用，会对其中一个机房进行下线升级，升级完毕之后，该机房上线，再对另一个机房下线进行同样操作。
