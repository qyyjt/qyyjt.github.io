    页面升级规范

    目的：自动化处理项目打包内网发布，减少外网发布发生的问题

    操作：

    # 1、 在项目中添加gulp脚本，参考

    https://192.168.100.188/svn/FaTeam/Demo/tags/mobile/vue-scaffold/gulpfile.js (opens new window)gulp手册： https://www.gulpjs.com.cn/

    必改项目： 项目打包上传的svn地址（内网，bugFix，外网） 内网web服务器上传地址 默认递交注释

    其它请按项目要求修改

    # 2、添加jenkins任务：

    jenkins服务器： http://10.15.97.192:8090/ 操作指南：

    点击新建

    输入项目名（同svn目录名），可以根据已有项目建立新任务

    修改描述信息，svn地址等

    # 3、页面开发完成后，源码上传svn。由jenkins进行打包上传服务器。必要时，点击任务后的执行按钮，立即编译。

    # 4、需求测试内网验证后，从内网webcontent的svn上将代码取出递交到release环境上。（同test，uat环境）

    注：禁止npm run build（yarn build）后直接上传release。
