# 项目开发流程

  1. 项目分配后，会在禅道上新建一个任务 (opens new window)给你，分配到每个人的任务、bug 都可以在禅道上查看 。

任务描述右侧会有相关需求，需求里面会有需求描述、 摹客地址和附件，附件中的原型图和设计逻辑需要下载下来查看；需求熟悉之后可以联系产品组织需求评审。

禅道上的任务在开始后都要设置截止时间，第一次提测后就点完成；如果需求没有上线，有变更的任务就激活，再设置完成时间；如果上线了，就再新建任务

修复好 bug 点击解决时，需要添加备注，说明问题原因、解决方法、影响范围等

  1. 依据项目脚手架或者空模板新建项目；如果是之前已经存在的项目，可以用 finalshell 监听页面路径来找到项目的地址，然后在原项目中修改。

  2. 项目初次完成后，可以更新到 dev 环境 服务器上查看并自测

  3. 对需求测试与 ui 测试后的反馈 bug 进行修复（分清 bug 和新需求。如果是新需求的话，请让需求在对应的 禅道 上进行添加）

  4. test 环境测试，修复禅道上的 bug。

  5. 项目发布后，将源码上传至 线上版本 svn

注意： 每次提交 svn 或 git 时，请按照格式填写详细功能修改记录

# 项目开发注意事项

  1. 项目开发中，发现问题请及时反馈

  2. 与原生交互处理： 原生方法放在 try...catch 中 老版本 app 请做兼容处理（咨询需求，老版本如何处理）

  3. 项目中索引页 index.html 中添加 项目包含页面列表，所需参数意义， 项目信息等

  4. 源码请及时上传，特别是多人合作开发项目

  5. 页面所用图片请进行压缩，在 ui 认可的情况下尽可能压缩

  6. 使用第三方库时，请注意移除无用的语言包

  7. 部分 android 自适应后 7.5rem 不代表 100% 宽度可以用百分比

  8. app 请不要使用 post，app 会拦截请求并丢弃 formdata

  9. 附件下载时，android 需要添加参数 Enclosure2018=true 作为附件依据

  10. zepto 的 on 委托事件不要放在 body 上

  11. iphonex 需要对安全区域做处理

  12. 请及时处理禅道上的 bug

  13. 项目完成请及时在禅道中的任务里点击“完成”，并备注项目源码地址。

# 日常工作

  1. 打卡

每天上下班需要打卡，可以通过财汇家园 - 考勤查询来查看每日的打卡情况；如果发现有遗忘打卡，可以通过填写遗忘打卡单，并交给人事来处理。

  1. 工作日志

工作日志是记录每日的工作内容和工作进度，需要发送邮件，具体格式可参考其他人邮件中的格式

  1. 加班

加班是从 18:30 开始，时长是 1 小时起步，半小时为 1 个单位进行时长叠加加班单填写流程：

填写纸质加班单（可以找旁边同事或者组长去拿，写完后统一交给组长） 填写电子加班申请 (opens new window)

如果你是最后一位加班，需要发送当天的加班申请邮件 别忘了发送当天的工作日志和下班打卡~

  1. 请假

如果有事请假，需要和领导申请，在得到领导同意后再进行请假流程：

填写纸质请假单（同样可以找旁边同事或者组长去拿，写完后交给前端组长杨丽博） 发送请假邮件 发送工作日志 别忘记下班打卡
