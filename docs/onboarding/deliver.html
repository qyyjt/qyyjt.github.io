# 一、终端PC

# 1、本地开发与打包

开发过程中可以将本地页面开发地址提供给需求查看，但上传后必须确保升级成功在通知需求验证。

升级容易出现问题，如自动打包时加密未通过，打包脚本服务器被重启。

Dojo项目请安照文档打包。

# 2、src.js中的依赖务必不能省略

# 3. F9项目目树添加节点

F9项目历史原因，部分页面由php实现。原目录树数据由php实现，现在目录树数据由前端页面提供。所以f9目录树修改是，前端页面要做对应修改。

# 7. 行情全量

getStockInfo: function (stocks, fields) {
  let defaults = "obj,name,new,lastclose,avgprice";
  fields = fields || defaults;

  return $.send("/objinfo/getinfo", {
    field: fields,
    where: "obj=" + stocks  // obj=SH600000.stk,SH600001.stk
  }, {
    before: function (result) {
      return JSON.parse(JSON.stringify(result).replace(/\bNULL\b/g, ''));
    },
    after: function (result) {
      if (fields == defaults) {
        result.forEach(o => {
          let price = parseFloat(parseFloat(o.new).toFixed(4));
          if (o.name == '--') o.name = '';
          o.avgprice = parseFloat(parseFloat(o.avgprice).toFixed(4));
          o.lastclose = parseFloat(parseFloat(o.lastclose).toFixed(4));
          o.new = price == 0 ? o.lastclose : price;
        })
      }
      return result;
    }
  });
},

# 二、企业预警通

# 1 做自适应处理

查见项目中 utils/dom.js resize方法

部分android 自适应后 7.5rem不代表100% 宽度可以用百分比

# 2 最细边框处理

查见项目中 sytles/mixin.less

# 3 与原生交互

原生方法放在try...catch 中，老版本记得做兼容 !important

原生文档： https://192.168.100.188/svn/FC_APP/开发相关/配置/原生跳转URL.xlsx (opens new window)页面整理文档：

项目脚手架中 utils/appAssist.js

appVersion android需测试, ios 页面自身跳转后消失

# 4 项目上传服务器设置

  1. 项目中包含gulp脚本（gulpfile.js）

注： gulpfile新项目中的几个地址需自行修改

gulp中task介绍：

build 删除打包文件夹并重新打包

checkout 下载内网打包后的svn项目

checkin 将打包后的文件复制到前面checkout下的文件夹后，强制checkin

upload 将打包后的项目上传至内网web服务器

  1. 项目传内网后，记得上传到： https://192.168.100.188/svn/jso/finchinaAPP/trunk/WebContent (opens new window)

  2. 项目发布前打包上传到： https://192.168.100.188/svn/jso/finchinaAPP/release/WebContent

uat、test、线上发布前环境

  1. 项目发布测试后，源码上传到： https://192.168.100.188/svn/FaTeam/Demo/branches/mobile_release

线上源码备份

  1. 线上bug修复，参见 新员工手册.docx

# 5 项目建立索引文件

index.html （项目包含页面列表， 项目信息等）

# 6 项目打包后文件过大处理

  1. 使用webpack-bundle-analyzer分析各个模块大小。

  2. 依赖的第三方库最小化，按需引入、去语言包等

  3. 尝试按需引入脚本

# 7 项目中链接的特殊处理

android 附件要进行处理 下载地址添加 Enclosure2018=true 如果是图片 需添加 imageAttachment=true

Iso 附件 需要添加 attachmentName=附件名

# 8 项目中图片处理

页面使用图片要进行压缩，可以压缩多次

上传到cdn的页面，不管压缩后图片大小，转成data image放入html中。

# 9 项目中使用的默认值处理

开发中默认值在打包时移除（gulp）

# 10 项目接口请求

本地开发过程中，url上使用万能token=000000，user任意

App中，会对页面请求进行拦截。将user与token塞入请求head里。

由于原生问题。请不要使用post方法，原生拦截时会丢弃formData。确定要使用post的话，请将参数加到url后。

# 11 要注意的ui与事件

input框在ios上显示有阴影， appearance: none;

zepto的on委托事件不要放在body上

iphonex适配 viewport添加viewport-fit=cover，内容区需要添加顶部和底部安全区填充，env(safe-area-inset-top)等，详见app_f9中的menu

分页参数统一 skip/pagesize, 如果不是务必于后台确认
